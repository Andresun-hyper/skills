## 项目名称：Demo Video Skill for CodeX

## 项目简介

这是一个面向 CodeX 工作流的本地化 Demo 展示视频生成 Skill，主要用于把网页原型、前端项目、交互式 HTML 页面或轻量化产品 Demo，整理成适合展示、汇报、比赛提交和作品集呈现的视频内容。

它的功能是录制已经可以运行的交互原型。

Overview

This project is a local-first demo video generation skill designed for CodeX workflows. It helps convert interactive web demos, frontend prototypes, and HTML-based product presentations into structured showcase videos for portfolios, design reviews, competition submissions, and product communication.

## 项目定位

这个仓库里的 Skill 更偏向“展示自动化工具”，适合已经做出网页 Demo、交互原型、可运行前端页面的人使用。它不负责凭空生成完整产品内容，重点在于理解已有 Demo 的结构，然后基于页面状态、交互路径和展示重点，自动生成更适合观看的视频化输出。

简单说，它做的是“把你的 Demo 包装成更清晰的演示过程”，让观众在更短时间内理解你的产品逻辑、核心功能和设计亮点。

## 使用原理

这个 Skill 的底层逻辑可以分成几个连续步骤。

第一步是识别 Demo 结构。它会读取本地网页或前端项目的页面层级、关键状态和主要交互节点，尝试判断哪些内容适合作为视频中的重要镜头，比如首页、功能入口、参数变化、动态反馈、结果页、提示状态或者特征动画。

第二步是组织展示流程。很多人在录 Demo 时会直接随手操作，导致视频节奏混乱、重点分散。这个 Skill 会把页面浏览顺序和交互顺序重新整理成更适合观看的展示路径，例如开场、功能引入、核心操作、亮点展示、结果反馈和收尾画面，从而让输出更接近完整演示视频的逻辑。

第三步是执行页面捕捉。Skill 会配合浏览器自动化或页面控制逻辑，在指定节点执行录制、截图、停顿、滚动、点击或状态切换，尽量避免人工录屏中常见的晃动、节奏不稳和重复操作问题。

第四步是整理输出结果。它会把录制到的片段按照预设顺序组织起来，并根据项目需求补充基础节奏控制，例如镜头长短、转场衔接、开头结尾保留、重点画面停留时间等。这样即使源 Demo 本身很简单，最终也能生成更利于展示的结果。

## 适用场景

这个 Skill 很适合用于设计作品集中的交互 Demo 展示，也适合比赛投稿、课程汇报、毕业设计展示、产品概念说明、网页功能演示和前端项目说明。只要你的内容本质上是“一个能运行的页面”，并且你希望别人快速看懂它的逻辑，这个工具就有价值。

如果你的项目已经具备明确的视觉风格和交互流程，它的效果会更好，因为 Skill 能基于现有结构提取更清晰的展示主线。

## 技术思路

这个项目采用本地优先的方式搭建，整体思路偏轻量和可控。核心能力通常围绕几个部分展开：页面访问与自动化控制、交互流程编排、录制或截图采集、素材拼接与导出，以及少量配置化参数控制。

在页面控制层，通常会使用浏览器自动化工具来驱动本地页面，例如打开目标地址、等待页面加载、触发点击、输入、切换状态、滚动页面或执行脚本。这一层的目标是稳定复现你希望展示的交互过程。

在录制层，可以根据不同实现方式选择屏幕捕捉、页面帧采样或外部录制方案。对于大多数前端 Demo 来说，本地浏览器自动化加录屏流程已经足够完成较高质量的展示视频制作。

在后处理层，会对捕捉到的镜头进行简单组织，例如统一时长、去掉无效停顿、补充首尾段、输出成适合进一步剪辑或直接投放的文件。

## 技术栈

主要技术栈为 Node.js、TypeScript、Playwright / Puppeteer 与 FFmpeg。Node.js 用于任务编排与本地执行，TypeScript 用于提升脚本可维护性与扩展性，Playwright 或 Puppeteer 用于浏览器自动化和页面交互控制，FFmpeg 用于视频片段处理、拼接和导出。

## 输入与输出

这个 Skill 的输入通常是一个本地 HTML 文件、一个可运行的前端项目地址、一个本地开发服务器页面，或者一组具有明确跳转关系的网页状态。

用户可以额外提供一些展示偏好，例如视频时长、重点功能、镜头节奏、是否突出交互细节、是否偏向作品集风格或比赛展示风格。这些信息会影响最终的录制顺序和输出节奏。

输出内容通常包括结构化镜头流程、录制片段、基础整理后的视频文件，或者供后续继续剪辑的中间素材。

## 项目特点

这个 Skill 的优势在于本地化、轻量化和可控性强。它不依赖复杂的第三方视频生成服务，也不要求额外接入重型云端 API，更适合已经有 Demo 的设计师、学生和开发者直接使用。

它尤其适合那些需要频繁迭代展示视频的人。因为一旦页面结构固定下来，很多录制动作都可以重复复用，后续更新版本时只需要重新执行流程，就能得到新的视频内容。
